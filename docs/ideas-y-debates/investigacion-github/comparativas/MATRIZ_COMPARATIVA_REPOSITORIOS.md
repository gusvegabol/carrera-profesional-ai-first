## Propósito

Esta matriz permite comparar repositorios que abordan una misma necesidad o
área funcional. No sustituye las fichas detalladas y no constituye por sí sola
una decisión de integración.

## Criterios de puntuación

Puntuar cada criterio de `0` a `5` y justificar las diferencias relevantes.

- **Encaje funcional:** relación con una necesidad real de Carrera AI-First.
- **Reutilización real:** posibilidad de aprovechar componentes separables.
- **Calidad arquitectónica:** claridad, modularidad, pruebas y adaptación.
- **Madurez:** documentación, actividad y estabilidad observable.
- **Dependencias:** simplicidad y control de las dependencias necesarias.
- **Licencia:** claridad y compatibilidad con el uso previsto.
- **Coste de integración:** esfuerzo técnico y operativo estimado.
- **Riesgo de desvío:** probabilidad de añadir complejidad innecesaria.
- **Valor de aprendizaje:** utilidad aunque no se integre código.

En coste, dependencias y riesgo, una puntuación alta significa menor coste,
menor dependencia o menor riesgo.
