# 06_ejemplos

Esta carpeta sirve para mostrar ejemplos concretos de buena y mala práctica, siempre con intención didáctica o calibradora.

## Convención de nombres

Los documentos de esta carpeta deben usar la raíz `EJEMPLO_`. Ejemplo: `EJEMPLO_TRANSICION_PROFESIONAL_RECONSTRUIDA.md`. La convención completa de nombres está definida en el README.md raíz de la bóveda.

Los pilotos reales o semi-reales no deben vivir aquí como documentos principales; deben ir en `../08_entrevistas_piloto/`.

## Función

- Aterrizar conceptos.
- Ilustrar decisiones de diseño.
- Hacer visible la diferencia entre profundidad y superficialidad.
- Servir como calibradores, no como pilotos reales.

## Contenido futuro

- Ejemplos de etapas profesionales.
- Ejemplos de buenas preguntas.
- Ejemplos de malas preguntas.
- Ejemplos de competencias con evidencia.
- Ejemplos de validación con el usuario.

## Relación con el playbook

Los ejemplos ayudan a comprobar si el playbook funciona en situaciones representativas.
Los pilotos reales o semi-reales deben ubicarse en `../08_entrevistas_piloto/`.
